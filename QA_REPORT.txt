Granpa Pump Selector - QA Summary

Version: 2.0.0
Project type: Native Android Java app. No WebView and no HTML wrapper.
Data source: app/src/main/assets/pumps.json generated from texmo_3_inch_borewell_submersible_pumpsets_page43_added.xlsx.

Data checks
- Pump records in package: 1780
- Pages covered: 1-43
- Missing pages: none
- Flow converted internally to LPH
- Catalogue section text is generated from page-header catalogue data

UI / UX checks
- Main screen top spacing improved to avoid status-bar cut-off.
- Main search box kept near the top for easier typing.
- Water mode, unit, pump type and phase dropdowns use custom two-line readable layout.
- Pump type dropdown has clear main categories and sub categories.
- Buttons use clearer hierarchy: orange primary action, blue catalogue/download action, muted secondary action.
- Results cards keep dealer format: model, HP/kW/phase, type, estimate, page/size/brand.
- Model details keep full technical format with page number.
- Catalogue section uses compact text block from data.
- Share image hides page number and sheet name.
- Share image chart spacing checked to avoid label/legend/footer overlap.
- QA screen added with logic summary.

Logic checks
- Fixed head only.
- No 10% upper limit.
- Fixed flow mode requires estimated flow >= required flow at fixed head.
- Range mode normalizes min/max and accepts reverse input.
- Search ignores spacing, so ACS1125 matches ACS 1125.
- Selection uses interpolation inside catalogue curve head range.

Build workflow checks
- GitHub Actions YAML has no merge conflict markers.
- Workflow uses Java 17 and Gradle 8.9.
- Artifact path points to app/build/outputs/apk/debug/*.apk.

Limitations
- Online product-image fetching is not enabled by default because the app is designed to work offline. The data model includes imageUrl for future mapped product images.
